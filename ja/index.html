Teradataを始めましょう

データサイエンティスト データエンジニア 開発者 データベース管理者 IT運用

オブジェクト ストレージに格納されたクエリー データ

Native Object Storage(NOS)は、AWS S3、Google GCS、Azure Blob、オンプレミス実装などのオブジェクトストレージのファイルに保存されたデータをクエリーできるVantage機能です。これは、Vantageに取り込むためのデータ パイプラインを構築することなく、データを探索したい場合に便利です。

Teradata Parallel Transporter (TPT)を使用した巨大なデータのバルクロード

Vantageに大量のデータを移動させるニーズはよくあります。Teradataは、このようなニーズにこたえるためTeradata Parallel Transporter (TPT)ユーティリティを提供しています。このハウツーでは、TPT の使用方法を説明します。このシナリオでは、30万件以上のレコードをもつ40MB以上のデータを数秒でロードします。

Teradata Vantageでdbtを使用する

このチュートリアルでは、Teradata Vantageでdbt (Data Build Tool)を使用する方法を説明します。オリジナルのdbt Jaffle Shopチュートリアルをベースにしています。いくつかのモデルは、VantageでサポートされるSQLに調整されています。

JDBCを使用してVantageに接続する

このハウツーでは、サンプルJavaアプリケーション(https://github.com/Teradata/jdbc-sample-app) を使用して、JDBCを使用してTeradata Vantageに接続する方法を示します。

VantageでMLモデルをトレーニングする

機械学習モデルのアイデアをすぐに検証したい場合があります。モデル タイプを念頭に置いています。まだMLパイプラインでの運用はしたくありません。念頭に置いていた関係が存在するかどうかをテストしたいだけです。また、本番環境のデプロイでも、MLops による継続的な再学習を必要としない場合もあります。このような場合は、Vantage Analytics Library(VAL)と、それがサポートする複数のMLモデルタイプを使用できます。
